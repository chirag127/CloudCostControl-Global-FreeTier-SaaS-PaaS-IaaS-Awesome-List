**CloudCostControl** is an indispensable, community-driven "Awesome List" meticulously cataloging global free-tier offerings across SaaS, PaaS, and IaaS providers. Designed for DevOps engineers, Site Reliability Engineers (SREs), and Cloud Architects, this resource empowers professionals to strategically minimize operational costs, streamline infrastructure provisioning, and accelerate development workflows by leveraging the best of the cloud's free ecosystem.

This repository serves as a definitive guide to free-tier cloud services, ensuring that users can efficiently discover, evaluate, and integrate cost-effective solutions without compromising on quality or scalability. From fundamental compute instances to advanced serverless functions and specialized databases, CloudCostControl aggregates vital information, enabling intelligent resource allocation and fostering innovation within budget constraints.

## ğŸ› ï¸ Getting Started

To explore and contribute to CloudCostControl, follow these simple steps. While this is primarily a content repository, basic setup allows for local validation.

### Prerequisites

Ensure you have the following installed:

*   **Git:** For cloning the repository.
*   **Node.js & npm (or Yarn/pnpm):** (Optional, but highly recommended for running linting and link-checking scripts locally.)

### Local Development Setup

1.  **Clone the repository:**
    bash
    git clone https://github.com/chirag127/CloudCostControl-Global-FreeTier-SaaS-PaaS-IaaS-Awesome-List.git
    cd CloudCostControl-Global-FreeTier-SaaS-PaaS-IaaS-Awesome-List
    

2.  **Install dependencies (if using Node.js for tooling):**
    bash
    npm install
    # or pnpm install
    # or yarn install
    

### Project Scripts

This repository includes a `package.json` with scripts for maintaining content quality (assuming `npm install` was run).

| Command            | Description                                                |
| :----------------- | :--------------------------------------------------------- |
| `npm run lint:md`  | Runs `markdownlint-cli2` to check Markdown formatting.    |
| `npm run check:links` | Validates all external links in Markdown files for reachability. |
| `npm run format`   | Formats all Markdown and relevant files using Prettier.    |
| `npm test`         | Executes all content quality checks (linting, link validation). |
